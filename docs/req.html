Objetivo


Este documento tem como objetivo mostrar para o cliente o significado por trás de jargões e termos técnicos utilizados no processo de construção do produto, afim de propiciar um ambiente claro e consciente do que cada funcionalidade exerce dentro do sistema.

Citar as principais funcionalidades do produto, através dos seus Requisitos Funcionais, bem como seus níveis de recorrência, criticidade e prioridade.

Mostrar o que há por trás do funcionamento dos requisitos funcionais, listando os Requisitos Não-Funcionais do produto, bem como suas respectivas descrições e métricas que irão tangir se estão sendo alcançados conforme projetados.

Explanar o funcionamento do produto no que se refere às Regras de Negócio e citar as medidas que o norteam, bem como Leis, Regras Contratuais, horários e dias de funcionamento, etc.

Mostrar a forma de contornar adversidades quando das mesmas.

Apresentar a descrição de cada indivíduo operador do produto. Citar características e ações oriundas de cada agente quando utiliza-lo e orientar a respeito de limitações e restrições estabelecidas a cada ator.

Instruir o cliente para evitar que situações adversas aconteçam, a fim de obter-se o melhor desempenho do produto e uma maior satisfação do cliente. Descreve-se como se faz o correto uso.






Glossário


  • Usuário - Personificação abstraída e individualizada de comprador e vendedor
  • Comprador - Ator da atividade de compra
  • Vendedor - Ator da atividade de venda
  • App - Sistema em questão a ser utilizado pelos usuários
  • Produtos - Mercadoria disponibilizada pelo vendedor para a atividade de compra do comprador
  • Carrinho - Montante de produtos selecionados pelo comprador, a fim de calcular o preço total da compra.

Descrição dos Atores

  • Usuário

    Personificação abstraída e individualizada de comprador e vendedor.

  • Comprador

    Ator da atividade de compra. Um dos alvos dos idealziadores deste produto. Perfil de usuário que movimenta o produto com a atividade de compra.

  • Vendedor

    Ator da atividade de venda. Um dos alvos dos idealziadores deste produto. Perfil de usuário que movimenta o produto com a atividade de venda.




Descrição dos casos de uso

  • Cadastrar Produto

    Aqui, permitimos que o usuário possa realizar cadastro de Produtos no sistema, permitindo que estes sejamdisponibilizados para compra. É realizado um registro no Banco de Dados, utilizado para que, quando requisitado, caso esteja em estoque, seja possível adicionar ao carrinho.

  • Cadastrar Usuário

    Aqui, permitimos que o usuário possa realizar cadastro no sistema, viabilizando o uso de suas diversas funcionalidades. É realizado um registro no Banco de Dados, utilizado para que, mediante login com id de usuário e senha, seja possível utilizar o produto.

  • Cadastrar Pedido

    Aqui, após selecionados todos os produtos desejados e adicionados ao carrinho, é possível formalizar uma requisição indivídual contendo todos os itens agrupados neste carrinho.

  • Cadastrar Pedido Item

    Aqui, de forma análoga ao item anterior, formaliza-se um pedido, no entanto, este refere-se ao item de forma individual, viabilizando que o cliente possa mencionar quantidade tanto para adicionar, quanto para retirar do carrinho, se assim desejar.

  • Realizar Login

    Aqui, o usuário deve fornecer os dados de acesso(Login e Senha). Após, verifica-se no Banco de Dados a autenticidade dos valores fornecidos, permitindo acesso ao sistema meditante autenticação bem sucedida, ou redirecionar para a tela inicial de login quando não autenticadas as informações.




  • Listar Vendedor

    Aqui, um comprador, após selecionar o produto desejado, pode visualizar todos os vendedores do tal produto em forma de lista, afim de selecionar o vendedorde seu agrado.

  • Listar Produto

    Aqui, um comprador visualiza de forma seletiva as variedades de produtos existentes.

  • Selecionar Produto

    Aqui, faz-se a seleção de um item para que seja marcado como item a adicionar no carrinho.

  • Adicionar ao Carrinho

    Aqui, um botão é exibido permitindo a inserção de um produto slecionado no carrinho.

  • Calcular Total

    Aqui, um botão dentro do carrinho é exibido viabilizando ao usuário o valor total de sua compra, de acordo com os produtos existentes em seu carrinho em determinado momento. Este valor é alterado toda vez que é feita um inserção, remoção ou mudança na quantidade de itens.

  • Finalizar Compra

    Aqui, mostra-se um botão de confirmação de requisição de compra dos itens adicionados no carrinho. Neste momento, são decrementados do estoque de produtos de determinado vendedor a quantidade de produtos requisitados por seu comprador.

  • Escolher Forma de Pagamento

    Aqui, o usuário tem botões Radio para escolher a forma de Pagamento(Débito, Crédito ou dinheiro).

  • Notificar Vendedor

    Aqui, quando há uma requisição de um pedido, existe um botão de sinalização da parte do comprador, para que o vendedor saiba que produtos dele foram requisitados.






  • Confirmar Entrega

    Após realizada a transação, realizado o pagamento e entregue a mercadoria, o vendedor e o produto sinalizam um pro outro, através de avaliação, mostrando satisfação com a compra e que foi finalizada a transação.
